# 时光绿径待办 · 产品介绍

> 微信小程序，搜「时光绿径待办」即用，无需下载安装。

---

## 一句话

一个微信里直接跑的待办工具，能单用、能组队，离线也能记。

## 功能亮点

**待办管理**
- 创建、编辑、删除、拖拽排序、滑动标记完成
- 标签分类（系统标签 + 自定义标签），按标签筛选
- 支持设置日期、时间、位置、图片、子任务、优先级
- 语音输入（微信语音识别插件）

**组合系统**
- 待办归入文件夹式「组合」，树形结构
- 组合内成员协作，三级权限：owner / admin / member
- 共享待办三种完成模式：全员完成、任一完成、指定某人完成
- 协作留言评论

**同步**
- 离线优先：先写本地 Storage，有网再同步到云端
- 增量同步，不是每次全量拉取
- 冲突检测：本地和云端同时改了同一件事，比时间戳合并，不改哪一方覆盖谁

**日历**
- 有待办截止日期的日期打点标记
- 点日期看当天待办列表
- 集成 @lspriv/wx-calendar

**统计**
- 完成率、趋势图、标签分布
- ec-canvas 图表

**打卡**
- 每日签到，连续签到有徽章
- 积分排行榜

**社区**
- 帖子发布、点赞、评论回复、@用户
- 举报处理

**通知**
- 微信模板消息推送，快到截止时间自动提醒
- 订阅管理

**分享**
- 待办生成分享快照（带密码保护）
- 访客记录、撤回

**工具集**
- 密码生成器、随机饮食、一言/毒鸡汤、星空效果
- 回收站、数据管理

**管理后台**
- 用户管理、额度调整、徽章管理
- 公告/更新日志发布
- 多维统计看板（留存率、标签使用、通知送达率、时段分布等）

## 技术构成

| 层 | 选型 | 备注 |
|---|---|---|
| 前端 | 微信小程序原生（WXML + WXSS + JS） | 无框架依赖，包体积可控 |
| UI 组件库 | TDesign Miniprogram | 腾讯出品，微信兼容好 |
| 后端 | Node.js + Express | 单进程，部署简单 |
| 数据库 | MySQL 5.5.62 | 老版本，不支持 JSON 列 |
| 服务端 | 阿里云 ECS 2C2G | 低成本运行 |
| Web 版（开发中） | Vite + Vue 3 + TypeScript + Pinia | 与小程序共用 API |

**依赖**: 微信登录、微信模板消息、WechatSI 语音插件

---

## 和同类工具有什么不一样

| 对比点 | 时光绿径待办 | 多数待办 App |
|---|---|---|
| 安装 | 微信里直接搜，不占桌面 | 要下 App 或打开网页 |
| 协作 | 组合内多人协作，三种完成模式 | 大多是单人工具 |
| 离线 | 离线优先 + 增量同步 + 冲突合并 | 纯在线或全量拉取 |
| 服务器 | 2C2G ECS 跑一年成本几百块 | 团队产品背后烧钱，说不准哪天停服 |
| 功能密度 | 待办+协作+日历+统计+社区+工具+后台 | 专注单点，但缺协同 |

## 不足（主动说）

- 没有暗黑模式（设计系统纯浅色，要加改动量大）
- 通知重试机制未实现（一次推送，失败不重试）
- 图片上传用图床，60 天无访问自动清理（服务器存储有限）
- MySQL 5.5 限制多，部分 SQL 写法要绕路
- Web 版开发中，尚未上线
- 没有公开 API / 第三方集成
